# Contributing to JobRecruita

Thank you for your interest in contributing! Please follow these guidelines to ensure a smooth workflow and high-quality codebase.

## Getting Started
- Fork the repository and clone your fork.
- Create a new branch for your feature or fix.
- Install dependencies with `npm install` at the repo root.
- Copy `.env.example` files to `.env` in each app and fill in values.

## Development
- Use `npm run dev:web`, `npm run dev:backend`, and `npm run dev:mobile` for local development.
- Write clear, modular, and well-documented code.
- Follow the existing code style and naming conventions.
- Add or update tests where appropriate.

## Commit Messages
- Use clear, descriptive commit messages.
- Reference issues or PRs when relevant.

## Pull Requests
- Ensure your branch is up to date with `master`.
- Run `npm run lint` and `npm run typecheck` before submitting.
- Describe your changes and link related issues/PRs.
- One feature/fix per PR is preferred.

## Code Review
- Address reviewer comments promptly.
- Be open to feedback and suggestions.

## Security & Compliance
- Never commit secrets or credentials.
- Follow best practices for input validation, error handling, and logging.

## Documentation
- Update relevant README files and docs for new features or changes.

## Code of Conduct
- Be respectful and inclusive.
- See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) if present.

---
For questions, open an issue or start a discussion.
